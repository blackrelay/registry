Black Relay Registry v1.0.1 corrects release metadata after the first public Registry release flow. It keeps the first source-aware Registry release surface for public EVE Frontier world data: the Registry API server, importers, indexer commands, PostgreSQL schema, source artefact handling, OpenAPI contract and release verification assets.

# Highlights
- Added PostgreSQL-backed Registry schema and migrations for canonical public world data.
- Added Go commands for API serving, migration, imports, Sui indexing and export generation.
- Added source-aware entity, fact, relation, killmail, current-state and operations APIs.
- Added OpenAPI 3.1 contract files for the public `/v1` API.
- Added static-client import paths for reviewed enemies, universe rows, type rows and recipe evidence.
- Added Sui GraphQL event and object backfill with cursor resume and current-cycle defaults.
- Added Cycle 6 package coverage for rift events, `rift::Rift` objects and storage-unit inventory v2 deposit and withdraw events.
- Added semantic killmail resolution for player, enemy and unresolved actors.
- Added content-addressed source artefacts and JSONL snapshot comparison support.
- Added public export generation, verification and local/R2 publication commands.
- Added release files for Windows, Linux, macOS, FreeBSD, OpenBSD and NetBSD.
- Added signed checksum manifests using the Black Relay release PGP key.

# Security
- No known security fixes in this release.

# In This Release
- Direct `br-registry` server binaries are published for Windows, Linux and macOS.
- Full command bundles are published for Windows, Linux, macOS, FreeBSD, OpenBSD and NetBSD.
- Command bundles include `br-export`, `br-import`, `br-indexer`, `br-migrate` and `br-registry`.
- Every command bundle includes `README.md`, `LICENSE`, `openapi/registry.v1.yaml` and JSON contract files from `contracts/`.
- Release checksum manifests cover every direct binary and command bundle.
- `SHA2-256SUMS` and `SHA2-512SUMS` are signed with the Black Relay release PGP key.
- The release workflow prepends download badges and verification guidance then appends a generated changelog from Git history.

# Operator Notes

PostgreSQL is the working Registry database. Raw source artefacts remain evidence. Public exports are distribution snapshots rather than the canonical store.

The public Registry path defaults to the current cycle. Cycle 5 support was removed from the current public path because the available World API and identity evidence is incomplete.

Cycle 6 rift support is source-backed by Sui rows. Rift spawn and location-broadcast events derive `site` records, location-hash evidence and broadcast system/coordinate facts when the chain emits them. The Cycle 6 manifest also includes the `rift::Rift` object type so object backfill can preserve current hashed-location evidence.

Storage-unit inventory rows derive conservative item type placeholders and action relations for mint, burn, destroy, deposit and withdraw events including Cycle 6 v2 deposit and withdraw events. Static-client type imports are still required to turn raw inventory `type_id` values into reviewed item or material names.

World API import coverage in this release is limited to reviewed system and tribe snapshots. Static-client imports remain the source path for universe data, enemies, broad type rows and reviewed recipes.

Sui object scans can be limited by upstream provider range windows. Registry records those gaps as coverage state instead of treating them as completion proof.

# Compatibility

This release establishes the initial public contract for:
- command names and major modes;
- PostgreSQL migration ownership;
- OpenAPI `/v1` routes and schema names;
- JSON export contract files;
- signed release checksum manifests.

Future incompatible API or export changes should be recorded in `CHANGELOG.md`.
