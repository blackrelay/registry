Black Relay Registry v1.1.0 is the Cycle 6 public-scope correctness release. It tightens the current Registry surface around verified Cycle 6 evidence, repairs character and tribe discovery, and makes exported current-state rows safer for public API consumers.

# Highlights
- Enforced strict current-cycle public exports for the Registry read model.
- Removed legacy Cycle 5 and `all` public read paths from current public scope.
- Required Cycle 6 event evidence before character rows appear in current public character collections.
- Repaired current character rows by replaying object IDs referenced by Cycle 6 character events.
- Preserved public character item IDs while repairing object-backed metadata.
- Filtered public character search and current character exports so stale object-only rows no longer inflate counts.
- Filtered public tribe search and current tribe exports to Cycle 6 public tribe profiles.
- Restricted current tribe IDs to the Cycle 6 public player tribe range plus the known Clonebank identity.
- Normalised duplicate current tribe identities such as `tribe:stillness:<id>` and raw `<id>` into one current public row.
- Preserved World API tribe names, tags, descriptions and URLs where source evidence is available.
- Enforced current-cycle export filtering so R2 and D1 consumers receive the same current public scope.
- Refreshed FreeBSD CI support through the pinned GitHub Actions dependency update.

# Security
- No known security fixes in this release.

# In This Release
- `/v1/current/characters` now reflects event-backed Cycle 6 character creation evidence rather than every historical or object-only character row.
- `/v1/current/tribes` now reflects public Cycle 6 tribe profile evidence and excludes raw placeholders such as `Tribe <id>` and `NPC Corp <id>`.
- `/v1/search` and typed list routes use the same public character and tribe filtering rules as the current-state endpoints.
- Public exports now apply the same strict current-cycle scope as the API read path.
- Character repair can fetch or replay character objects that are directly referenced by Cycle 6 events, even when broad object-by-type scans are provider-window limited.

# Operator Notes

After upgrading from v1.0.1, rebuild current public exports before publishing them to edge storage or D1. The old export shape can contain historical or object-only character rows that are no longer part of the supported public current scope.

Recommended repair sequence:
```sh
br-indexer -mode events -cycles current -max-pages 0 -concurrency 64
br-indexer -mode derive-events -module killmail,character,gate,assembly,storage_unit,turret,rift -derive-batch-size 5000
br-indexer -mode repair-character-objects -derive-batch-size 5000 -concurrency 16
br-export -out ./exports -cycles current -include-events -include-sui-objects
br-export verify -dir ./exports
```

Windows:
```powershell
.\br-indexer.exe -mode events -cycles current -max-pages 0 -concurrency 64
.\br-indexer.exe -mode derive-events -module killmail,character,gate,assembly,storage_unit,turret,rift -derive-batch-size 5000
.\br-indexer.exe -mode repair-character-objects -derive-batch-size 5000 -concurrency 16
.\br-export.exe -out .\exports -cycles current -include-events -include-sui-objects
.\br-export.exe verify -dir .\exports
```

The public API distribution should rebuild D1 from a verified current export after this Registry upgrade. Incremental event and raw-index syncs are not enough to repair public current-state rows that were previously exported with looser scope.

# Compatibility

This release intentionally removes older-cycle public current reads. `cycles=current` and `cycles=6` are the supported public Cycle 6 options. Other cycle values should be treated as unsupported for public Registry reads.

The canonical PostgreSQL store can still preserve raw evidence rows that are outside the public current scope. Public exports and public API routes should not expose those rows as current Cycle 6 identities.

# Known Limits
- World API tribe profile availability can lag chain events. Registry only promotes source-backed public names, tags, descriptions and URLs.
- Sui object scans can still be limited by upstream provider range windows. Event derivation and event-referenced object repair remain the primary Cycle 6 repair path.
- Current public rows are conservative. If a row lacks enough source evidence for public identity, it should remain evidence in PostgreSQL rather than appear as a resolved public identity.
