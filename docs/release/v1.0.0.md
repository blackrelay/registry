Black Relay Registry v1.0.0 is the first source-aware Registry release for public EVE Frontier world data. It packages the Registry API, importers, indexer commands, PostgreSQL schema, source artefact handling, OpenAPI contract and release verification assets into one operator-ready repository.

## Highlights
- PostgreSQL-backed Registry schema and migrations for canonical public world data.
- Go commands for API serving, migration, imports, Sui indexing and export generation.
- Source-aware entity, fact, relation, killmail, current-state and operations APIs.
- OpenAPI 3.1 contract for the public `/v1` API.
- Static-client import paths for reviewed enemies, universe rows, type rows and recipe evidence.
- Sui GraphQL event and object backfill with cursor resume and current-cycle defaults.
- Cycle 6 package coverage for rift events, `rift::Rift` objects and storage-unit inventory v2 deposit/withdraw events.
- Semantic killmail resolution for player, enemy and unresolved actors.
- Content-addressed source artefacts and JSONL snapshot comparison support.
- Public export generation, verification and local/R2 publication commands.
- Release archives for Linux, Windows, macOS, FreeBSD, OpenBSD and NetBSD.
- Signed checksum manifests using the Black Relay release PGP key.

## Operator Notes

PostgreSQL is the working Registry database. Raw source artefacts remain evidence. Public exports are distribution snapshots rather than the canonical store.

The default cycle scope is the current cycle. Historical cycle imports require explicit cycle selection and may have unresolved names when the matching World API or static-client source artefacts are unavailable.

Cycle 6 rift support is source-backed by Sui rows. Rift spawn and location-broadcast events derive `site` records, location-hash evidence and broadcast system/coordinate facts when the chain emits them. The Cycle 6 manifest also includes the `rift::Rift` object type so object backfill can preserve current hashed-location evidence.

Storage-unit inventory rows derive conservative item type placeholders and action relations for mint, burn, destroy, deposit and withdraw events including Cycle 6 v2 deposit/withdraw events. Static-client type imports are still required to turn raw inventory `type_id` values into reviewed item or material names.

World API import coverage in this release is limited to reviewed system and tribe snapshots. Static-client imports remain the source path for universe data, enemies, broad type rows and reviewed recipes.

Sui object scans can be limited by upstream provider range windows. Registry records those gaps as coverage state instead of treating them as completion proof.

## Compatibility

This release establishes the initial public contract for:
- command names and major modes;
- PostgreSQL migration ownership;
- OpenAPI `/v1` routes and schema names;
- JSON export contract files;
- signed release checksum manifests.

Future incompatible API or export changes should be recorded in `CHANGELOG.md`.
